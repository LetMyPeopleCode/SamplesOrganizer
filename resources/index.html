Samples Organizer Help

Our help can be found by clicking one of the question mark icons around the app, or in our Github repository in the Docs folder.

Using The Folder Selector

Selecting "Open a samples folder" will open a file dialogue. Navigate to and select the directory you want to open.
 
The process will gather file information for all files in that folder and its subfolders.
CAUTION
If you open a folder with thousands of files in it and its subfolders, it could take a long time or possibly overload the app. If this happens, drill down to a subfolder that has fewer files.

Using The Search Form

Before we explain the form, we need to explain two ideas: AND and OR.
  • AND: All conditions must be matched.
  • OR: At least one condition must be matched

 

Each text field can take multiple values, separated by commas:
  • If you have multiple, they will be treated as an OR
  • File Name - is - Loki, Thor will come back with all files where the file name is exactly "Loki" or "Thor".

 

is, is not, contains, or does not contain
  • is, isn't: Require an exact match. is "Thor, Loki" will only match records where the value is Thor or Loki (exactly). isn't would match every record but those.
  • contains, doesn't contain: Requires a partial match. contains "Thor" would match "Thor" and "Thorn". doesn't contain would match every record but those.

 

Each information type is connected by an AND:
  • If you set "file name is 'Thor, Zeus'" and "tags is 'norse'", it would only match Thor, because both the name AND the tag must be present.

 

Tags has two lines: one with "contains" and one with "does not contain.":
  • If you want all the tags with an assigned BPM, you can just use Tags contains "BPM", but since many sounds without a known BPM are assigned an "Unknown BPM" tag, adding Tags does not contain "Unknown BPM", will exclude those.

 

The Search Results pane shows you the results of your search. To browse/edit any of those files use Open Result Set In Browse to switch back to the Browse tab and treat the search results like they're the contents of a directory.

About File Information

The File Information Area contains information fields about the file and controls for the browsing process. The fields contain information we guessed from the file path, or information you entered/changed. Here are their names and what they represent or do.

  • File Name: This can be the actual name on disk or you can change it to something more meaningful.
  • Source: The company or producer that made it.
  • License: This can be the path to a license file (which we'll autopopulate if we detect a good candidate), the name of a license (like CC-BY), or your own description.
    • Write your own text or select Choose to locate the license file on disk.
    • If a valid path is shown, you can select "view license file in folder (if path)" to open your file manager to the folder containing the license file.
  • Comments: If you have a specific comment you need to record, put it here. (max 1,000 characters - no markup)
  • Speed Browse: Check this box if you want to rapidly browse through a list of files generated by selecting a folder or searching the database.
    • S skips it (still in the database, but with a "skipped" tag so you can come back and review it later).
    • K keeps the active file and saves any tags or other information you added/
    • The keys above are disabled when you have your cursor in a text field like Comments. This helps make sure, if you're typing "I love this file", you won't accidentally skip it and move to the next file when you type the "s" in "skip"
    • Pausing a Speed Browse: All recognized sound files are stored in the database with a pending review tag when their directory is loaded for the first time. You can leave off and come back later by doing a search for the "pending review" tag.
    • You can speed browse any set of files, whether loaded from a folder or the results of a search, for example doing a second pass on files you kept. Choosing "keep" on a file you previously skipped will swap the "keep" and "skip" tags for that file. The same goes for a file you previously kept, but now want to skip.
    • If you're not speed browsing files, but you're going through them a bit more... "formally"... saving the sample information will remove the pending-review tag.
    • Speed browsing can be done on folders or file sets created by a search.

About Tags

Tags are short words or phrases that help you categorize a file. You might tag something with 120 bpm or funky or good for edm.

Their power lies in combining them. "Show me files tagged as orchestral, happy, and loop, that are shorter than 20 seconds.

The tags box is separated into three sections:

  1. Add/Search Tags: Start typing in this section to get suggestions of possible tags. If we suggest one you like, select it to add it. If none we suggest work, finish typing and select Add to add it.
  2. This File's Tags: These are tags we either guessed the first time you loaded a file or the ones you've assigned. To remove a tag, click it.

WARNING: Only lowercase US English letters (a-z), spaces ( ), dashes (-), and underscores (_) are allowed in tags. There's a mechanism that prohibits all others and converts capitals to lowercase. We may add support for symbols outside of US English like ñ and é in the future.
